语言及框架选型

发表于 2021-08-30
字数统计: 543 | 阅读时长 ≈ 1

语言选型

  1. python

    优点：原本语言基础、开发难度低，Django和Flask框架文档完善，Sanic等新的异步web服务器性能也ok。

    缺点：性能没有c++、java、go好

  2. go

    优点：性能、学习难度较低

    缺点：仍需要学习时间成本

  3. java

    优点：稳定、性能好、web通用方案，成熟度高

    缺点：学习时间成本较高

框架调研报告

框架名 语言 特点 50客户端下TPS 150客户端下TPS 1000客户端下
Django Python 功能全面，文档齐全，模板可以快速开发页面但是也与前后端分离的思想有点不符 4000 3000 1000左右
Flask Python 同步，轻量，微框架设计，原生只有路由和视图功能，但有丰富的三方插件可以快速实现WEB常用功能
Tornado Python 异步，较为原始，诸多内容需要自己去处理。适合不需要依赖太多框架自身功能且对性能要求较高的项目，异步机制是其自实现的方法，所以学习成本较高，Tornado项目代表：知乎
Sanic Python 异步框架，算比较新的一个框架，语法类Flask, 学习成本低，官方文档有中文维护版本
net/http Go 原生，异步

参考文档

https://www.jianshu.com/p/2de7546763ca

Tornado 的好成绩得益于其自带的异步特性，而 Django 与 Flask 是同步框架，在处理请求时性能受限。但是实际使用中，一般是 Django/Flask + Celery + Redis/Memchaned/RabbitMQ 的模式，由此带上了异步处理的能力。

  • Flask：微框架的典范，号称 Python 代码写得最好的项目之一。Flask 的灵活性，也是双刃剑：能用好 Flask 的，可以做成 Pinterest，用不好就是灾难（显然对任何框架都是这样）。Flask 虽然是微框架，但是也可以做成规模化的 Flask。加上 Flask 可以自由选择自己的数据库交互组件（通常是 Flask-SQLAlchemy），而且加上 celery +redis 等异步特性以后，Flask 的性能相对 Tornado 也不逞多让，也许Flask 的灵活性可能是某些团队更需要的。
